MCP Steroid

An IntelliJ MCP server that lets AI agents run Kotlin code inside the IDE, with vision, OCR, and human review gates. The server exposes the full IntelliJ platform: PSI for code understanding, refactorings, inspections, and project model queries.

Agents submit Kotlin scripts via steroid_execute_code, and a review gate allows human approval before execution. Vision tools capture screenshots with component trees and OCR, enabling agents to click what they see.

Homepage · GitHub Tags: mcp, intellij, ai-coding

Russian Loto with AI Voices

A fast prototype of a Russian Loto (Bingo) game announcer with 12 AI-generated celebrity voices. It started as a quick AI experiment and turned into great family fun. Each character has a unique personality and tells jokes for every number from 1-90.

The game uses Qwen3-TTS VoiceDesign to generate native Russian voices from text descriptions on a local Mac, which makes iteration fast with no voice samples. Characters include Brezhnev (Soviet leader gravitas), Monetochka (Gen-Z indie pop), Korol i Shut (punk rock horror), Vysotsky (legendary bard), and more.

Features:

  • 12 distinct Russian celebrity voice personalities
  • Character-specific jokes for all 90 numbers
  • Noise detection with exponential backoff (waits for quiet room)
  • Runs locally on Apple Silicon with MPS GPU acceleration
  • VoiceDesign model generates voices from text descriptions

GitHub · Blog Post Tags: ai, tts, python, qwen

Plugin Hot Reload

Hot reload IntelliJ plugins over HTTP without restarting the IDE. Upload a plugin ZIP to a local endpoint, and it unloads the old version, replaces files on disk, and loads the new descriptor—all while the IDE keeps running.

The plugin writes a discovery marker with a bearer token on startup. A Gradle task scans these markers and pushes new builds to running IDEs automatically, cutting the edit-compile-test cycle from minutes to seconds.

GitHub Tags: intellij, plugin-development, hot-reload

Roomtone

A tiny, self-hosted single-room call service with WebSocket control and WebRTC signaling. One room, one URL, no meeting sprawl. Designed for homelab use.

The server keeps in-memory room state and speaks WebSocket for join, leave, and signaling. A Telegram bot issues time-limited, RSA-signed JWT invite links for access control. Two media paths are supported: real-time WebRTC and WebSocket relay using MediaRecorder.

GitHub Tags: roomtone, webrtc, self-hosting

Stevedore

A lightweight GitOps-style deployment loop for Docker Compose on a single host. Push code to GitHub, and your Raspberry Pi updates itself. No CI pipelines to configure, no complex Terraform state to manage.

The daemon polls enabled deployments for changes and runs docker compose up -d --build. It handles failed builds gracefully, manages secrets in an encrypted SQLCipher database, and can update itself. Designed for the “fork first” philosophy—your infrastructure, your control.

Stevedore DynDNS is a companion service that keeps DNS, HTTPS, and routing in sync. It updates Cloudflare DNS when your IP changes, obtains wildcard certificates via Caddy, and routes subdomains to internal services automatically.

Stevedore GitHub · DynDNS GitHub Tags: stevedore, gitops, raspberry-pi, docker, dyndns

AudioBroadcastr

I was using Windows that time and Spotify. I also had Pioneer audio AV-receiver connected to home network. I wanted to listen Spotify via it. So the goal was to capture all playing sounds from by PC and stream it to the AV-receiver.

I created a tiny (and ugly) HTTP server, found the way to force AV-receover to play an HTTP .mp3 stream form my server. Made use of .mp3 library to encode what-u-head sound stream into an .mp3 stream. Found a way to capture what-u-hear audio stream on Windows. Made a commandline utility out of it. That is a hackathon quality project that solved my need.

It turned out I used it only a few times, quite soon I switched to Mac OS X and started using Air Play for that.
